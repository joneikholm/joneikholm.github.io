How useCollection Works

Real-time Firestore listener — internally uses useReducer + useEffect

② useEffect (actual source, simplified)
useEffect(() => {
// if no query, clear & exit
if (!ref) { setValue(undefined); return; }
 
// subscribe to Firestore
const unsub = onSnapshot(
ref,
setValue,   // success → updates state
setError   // failure → updates state
);
 
// cleanup: stop listening
return () => unsub();
}, [ref]);
 
return [value, loading, error];
→
onSnapshot opens persistent connection to Firestore
→
setValue / setError write to React's slots → triggers re-render ③
→
Cleanup unsub() runs on unmount → closes connection
⟳
Any change in Firestore → setValue fires again → new re-render
② → ③ repeats on every Firestore change
