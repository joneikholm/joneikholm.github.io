Transaktionskommandoer

Java + SQL — det du skal bruge
Connection connection = DriverManager.getConnection(url, user, pw)
Opret forbindelsen til databasen. Denne bruges til alle kommandoer herunder.
connection.setAutoCommit(false)
Start en transaktion. Fortæl databasen: "jeg styrer selv hvornår ændringer gemmes."
connection.commit()
Gem alle ændringer. Alt siden setAutoCommit(false) bliver permanent.
connection.rollback()
Fortryd alle ændringer. Alt siden setAutoCommit(false) slettes, som om det aldrig skete.
connection.setAutoCommit(true)
Nulstil til normalt. Hver SQL-sætning gemmes automatisk igen. Brug i finally.
SELECT ... FOR UPDATE
Lås rækken. Andre transaktioner med FOR UPDATE må vente til du har lavet commit.
BEGIN / COMMIT / ROLLBACK
Ren SQL — samme princip uden Java. BEGIN starter, COMMIT gemmer, ROLLBACK fortyder.
Typisk mønster
connection.setAutoCommit(false) // start
  SELECT ... FOR UPDATE  // læs og lås
  UPDATE ...             // skriv
connection.commit() // alt gik godt → gem
connection.rollback() // noget gik galt → fortryd
connection.setAutoCommit(true) // altid til sidst (finally)
